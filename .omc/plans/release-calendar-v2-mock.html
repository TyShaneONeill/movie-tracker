⌄  Swipe down for week view

Expanded snap point for scanning far ahead. ‹ › move one month here. Tap any day → dock snaps back to week view centered on it, results swap above.

Interaction spec (for the build brief)

  • Dock = bottom sheet with two snap points (week strip / full month). Drag handle + spring animation; respects reduce-motion.
  • Arrows navigate the visible granularity: week-strip view → ±1 week (month label follows the week); expanded month → ±1 month.
  • Strip swipe pages by week. Day tap (either state) selects the day, swaps the results list, and — from expanded — snaps back down to week view.
  • Selected day is always visible in the strip (auto-center on selection or week change).
  • Data behavior unchanged: month-window fetching stays as-is (week strip is a view over the loaded month; crossing a month boundary triggers the existing month load).

Shared decisions (both variants)

  • Filters move from a hidden sheet to always-visible chips under the header — one tap instead of two, and state is glanceable. The ⚙ icon keeps the full filter sheet for the long tail.
  • Day header shows the count ("4 releases") so scanning between days has feedback even before the list settles.
  • Taste-match line reuses the existing personalization scores (top picks bubble up — Track B #1 relevance work slots in here).
  • Dot semantics kept: brand dot = releases, gold dot = watchlist titles. Selected day inverts.
  • Dock is a real bottom sheet: drag handle, spring to snap points. Calendar never scrolls away — it's pinned, per the roadmap note.
